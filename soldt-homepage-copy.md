**IMPLEMENTATION NOTES**

# **Dev Handoff Notes**

* All section headlines and body copy are final draft—ready for implementation. Adjust only for character limits in specific UI components.

* CTA button labels are suggestions. If your design system has a primary/secondary/ghost button hierarchy, map accordingly: first CTA in each section \= primary, second \= secondary.

* Placeholder screenshots should use the SOLDT purple (\#777DD3) as the dominant accent colour with near-black (\#09090b) for text. White backgrounds preferred for dashboard mockups.

* FAQ section should be a true accordion component (not a static list). Only one item open at a time for cleaner UX.

* Comparison table should be responsive: on mobile, either convert to a stacked card layout or a horizontally scrollable table. Don’t let it break.

* Blog cards should pull from the actual /blog CMS. The sample cards above are placeholders—replace with real posts once content is published.

* Email capture should integrate with your newsletter service (e.g. ConvertKit, Loops, or a custom Supabase function). Include honeypot field for spam prevention.

* Persona cards should link to filtered landing pages if available (e.g., /for/wig-sellers). If not, link to /signup with UTM tags for persona tracking.

* Trust badges: source official partner logos where applicable (PayFast verified seller badge, SSL cert badge). For uptime and POPIA, use custom SOLDT-branded badges.

* Mobile: every section must work at 375px width. Test the comparison table, persona cards, and email capture on mobile first.
